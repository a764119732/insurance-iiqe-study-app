## 1. 本轮目标

本报告用于 Risky 阶段后续规划：在排除已处理题、已审计题和禁止自动处理题后，对剩余风险题进行分层，并给第四批候选筛选提供依据。

本轮只读分析题库和既有报告，未修改 JSON、UI、PDF、离线版、`sharedFiles`、脚本或 README。

## 2. 统计口径

本报告使用“操作层风险池”口径，不等同于 `NEW_QUESTION_QUALITY_AUDIT.md` 中的原始 Risky 标签数 699。

操作层风险信号包括：

- 当前 `simple_explanation` 仍含“新增题待人工复核”模板。
- 题干包含否定或排除信号。
- 题干或选项为 i / ii / iii / iv 组合题。
- 选项包含“以上各项 / 以上所有 / 以上皆是”等全项判断。
- `original_explanation` 去空白后少于 20 字。

分层规则：

| 层级 | 口径 |
| --- | --- |
| Auto Rewrite Candidate | 仍为模板解释，`original_explanation` 不为空且不过短，未在禁用清单内 |
| Manual Review Required | 有风险信号，但当前不是模板解释，或需要先人工审计再决定是否可重写 |
| Do Not Auto | `original_explanation` 为空或过短，当前 JSON 不足以支撑自动重写 |

说明：Auto Rewrite Candidate 不代表可以立刻写入 JSON；仍建议按批次先做 `MANUAL_AUDIT`，再生成 `REWRITE_PLAN`。

## 7. 后续处理原则

### Auto Rewrite Candidate

适合继续按批次推进，但仍应先人工审计：

1. 每批先选 30 题生成 `MANUAL_AUDIT`。
2. 人工审计后只把依据充足的题进入 `REWRITE_PLAN`。
3. 写入时只改 `simple_explanation`。
4. 写入后必须做 `REWRITE_AUDIT` 和 `SPOTCHECK`。

### Manual Review Required

不应直接写入：

1. 先读取题干、选项、答案和原解析。
2. 若发现答案依据不完整、题干/选项语义风险、法律边界风险，转入禁止自动处理清单。
3. 若人工审计确认可重写，再进入下一批 `REWRITE_PLAN`。

### Do Not Auto

不应自动重写：

1. `original_explanation` 为空或过短。
2. 当前 JSON 不足以支撑解释重写。
3. 需要 PDF、教材或人工依据补充后再决定是否处理。
